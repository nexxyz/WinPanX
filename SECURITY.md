# Security Policy

## Supported Versions

Security fixes are provided on a best-effort basis for the latest released version.

## Reporting a Vulnerability

Please avoid opening public issues for security vulnerabilities.

Use one of these channels:

- GitHub Security Advisories (preferred): open a private report in this repository.
- If advisories are unavailable, open an issue titled `SECURITY: private contact requested`
  and do not include exploit details.

The maintainers will acknowledge valid reports and coordinate a fix/release window.
